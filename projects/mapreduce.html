Projects / MapReduce Framework

MapReduce-style framework

What it is

A Python MapReduce runner inspired by Google’s original MapReduce model: a coordinator schedules work across multiple workers, streams intermediate results, and re-runs tasks on failure. The design prioritizes reliability (fault tolerance) and throughput (parallelism).

Tech stack

  • Python
  • Threads + processes for concurrency
  • Socket networking
  • Structured logging for debugging

Highlights

  • Worker registration and heartbeats
  • Task scheduling with retries on failure
  • Map output partitioning and reduce aggregation
  • Deterministic behavior for testing

Why it matters

Distributed execution is easy to demo and hard to make dependable. This project demonstrates comfort with: coordination protocols, race conditions, and designing APIs that make it easy to plug in new MapReduce “jobs.”
